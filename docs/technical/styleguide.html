In general ELLA has very few headers, this stems from the use of *boxes to present context to the user. This pattern allows for very information-dense views without needing to have large, space wasting headers.

reserved headers

primary header

primary header - callout

primary header - less important

These are only found when representing top-level - user/sample/allele - information to the user.

sectionbox-header

contentbox-header

This style is reserved exclusively for the sideways headers on boxes and establish ELLA's core theme. Vertical bars with _very_ faded boxes are used to provide context to the user in a space efficient manner - they also can be thought of as representative of the bars in e||a. No content should be presented to the user in this form, only commonly repeated headers.

variant / acmg header

Another highly reserved style, always presented in this faded blue. Used to designate a major portion of a context - f.ex the sidebar or ACMG areas. Where one of the lighter headers doesn't call enough attention, or already has a context.

empty-state or informative text

This is used when an element is currently in an 'empty state' and no other information is being given to the user - or - in modals when the only information being given to the user is informative/help text and the buttons present clear options. It should never be paired with other text or elements due to its faded nature. It's meant to be helpful if noticed, but not the element demanding attention on the page.

sidebar/variant text - gene

sidebar/variant text - c.variant

Only used on the sidebar, has been left unique from other headers to help distinguish from other uses, and because of the small size needed.

common headers and text

Common text header:
common text

This text pairing is used in nearly all content displayed for the user; including tables and other headings (e.g. reference section)

common clickable text - either a link or a popup

Clickable elements text elements are presented with a dashed underline - this creates a uniform text decoration that can be used throughout the interface to inform the user without being overly intrusive

popover header

Used only as the title of popovers - should likely be merged with the header for question elements

popover/modal inner-header

Used as a header inside popovers and modals

table / minor header

Used as the header for table-columns as well as when a header could be useful to the user, but might not be needed. Because of this, it needs consideration whether or not to use it! An example is where it's used as a header for each reference, it might not be needed there if each reference can be distinguished.

elements

question-title
question-text

These styles are used heavily in popovers where the user is 'filling out forms'.

Buttons generally inherit the sectionbox's color, except on the navbar, in 'question' elements or where a callout is needed - f.ex green for 'start' or pink for 'warning'.

Note the 'normal' style can be used to force a button to not use the parent's coloring - this is typically done when another button is a callout such as an override.

Inputs and textareas share styling, and are characterized by their heavy underline and faded placeholder text.

Note that the placeholder text used is different from user-inputted text. User text uses Source Sans and is more dense.

ACMG - BP2

ACMG codes are one of the few standout elements that re-use very few themes from above. They have a bar on the left (a common theme in ELLA) to preserve context for the user - e.g. when comments are tall it's easy to tell which comment belongs to which code - but otherwise have unique coloring and styling. Because of their importance in the application this is probably a good thing. :)
